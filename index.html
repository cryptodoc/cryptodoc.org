CryptoDoc

Documents for Digital Era

Open source crypto container standard for smart contracts, data and documents.

Created for people and machines. Ready for Web, Blockchain and IoT. No more paper.

Pure

CryptoDoc based on concept of readable data and programatic purity. It's human and machines friendly.

  • People can read and edit documents.
  • Computers can read this data.
  • Blockchain free. It can be used by any blockchain.
  • Offline first. It can be used without access to global network.

Simple

Based on well known technologies.

  • Descriptive. CryptoDoc uses descriptive format to describe interfaces.
  • Embeddable. It's ready to be used on tiny devices.
  • Use existing code base. There are ready libraries for CryptoDoc components on C, Swift, Go, Rust, JS and other popular languages.
  • Vendor free. No fees, no patents.
  • You can create your own CryptoDoc today without using of any kind of special software except of digital signature app.

Powerful

CryptoDoc is a mix of several technologies with bullet-proof efficiency.

  • Allow to use files of any format.
  • Support multiple forms of signature: single, mutlti, collective.
  • Country-lock free. It can support multiple digital signatures: USA (FIPS), EU (CEF), RF (GOST).

Container

CryptoDoc is a container. It's pretty lighweight. It provides strict predictable data structure. It consist of:

  • Data
  • Signature

Data

CryptoDoc is a container. It's pretty lighweight. It provides strict predictable data structure. It consist of:

  • Data in JSON or XML
  • Photo or 3D model of face.
  • Any applicable data: image, audio, video, word, pdf.
  • Code, Smart Contract.

Signature

One of the most useful and powerful thing of CryptoDoc is signature. Data could be signed by one or multiple signatures. Each signature can be verified by chain of signatures.

  • Single, multiple or collective.
  • Personal or verified by other signatures.
  • Any of primitive: EDDSA, FIPS, CIP e-Sign, GOST.

Goal

The goald are to implement world-wide standard for digital documents based on principles of open source software and open governance and to rise ecosystem for software developers.

Status

Current status of project is development of MVP and initial codebase. We are working on the draft of standart.

Law

CryptoDoc is a container. So it can include signatures of any type and kind. Documents can be modified to be used in any country without any changes in standard.

Users and Usage

The users of this documents are governments, companies and groops of people. It can be used as contract or smart contract, diploma or even multi passport?

Progress

Currently we are working over specification draft and MVPs for viewing, storing and verifing documents. After draft will be completed we;re going to create libraries for creating experimental products and early adpoters to finalize requirements to 1.0 version.
